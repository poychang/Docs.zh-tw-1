# <a name="39ltemptyconstantgt39-is-not-declared"></a>&#39;&lt;emptyconstant&gt;&#39; 不宣告
'\<emptyconstant >' 未宣告。 不再支援空的常數。請改用做任何動作。  
  
 宣告或指派陳述式嘗試將值指派`Empty`到變數、 常數、 列舉型別成員、 屬性或函式傳回。  
  
 舊版的[!INCLUDE[vbprvb](~/includes/vbprvb-md.md)]用`Empty`關鍵字來表示未初始化的資料存放區。 目前版本的 Visual Basic 不支援`Empty`。 未初始化的變數會保留其資料類型的預設值。 如需預設值的詳細資訊，請參閱 < [預設值] 中[Dim 陳述式](../../visual-basic/language-reference/statements/dim-statement.md)。  
  
 [Nothing](../../visual-basic/language-reference/nothing.md)關鍵字代表任何資料類型的預設值。 您可以使用，而不是`Empty`。  
  
 **錯誤 ID:** BC30823  
  
## <a name="to-correct-this-error"></a>更正這個錯誤  
  
-   使用 `Nothing` 取代 `Empty`。  
  
     -或-  
  
-   使用適用於程式設計項目的資料類型的預設值。  
  
     -或-  
  
-   如果這是在變數宣告，請勿指派一個初始值。 這會導致要為其預設值初始化的變數。
